# Rejected Sources

Rejected sources are not necessarily "bad"; they were not strong enough for this page compared with available primary sources.

## 1. Generic "AI Agents Explained" Vendor Landing Pages

- **Examples searched:** broad IBM, Salesforce, Oracle, Accenture, and consulting pages explaining "what are AI agents?"
- **Type:** marketing / secondary overview
- **Evidence tier:** Reject
- **Reliability:** secondary
- **Reason rejected:** Useful for market vocabulary, but they rarely add technical details beyond stronger primary sources from Anthropic, OpenAI, Google, protocol specs, and papers.
- **Better source:** Anthropic Building Effective Agents; OpenAI Practical Guide; Wooldridge & Jennings.
- **Note:** skip

## 2. Medium and Substack Retellings of ReAct / Reflexion / AutoGen

- **Type:** secondary explainer
- **Evidence tier:** Reject
- **Reliability:** secondary
- **Reason rejected:** The original papers are accessible and more precise. Retellings often blur agent loops, workflows, and product platforms.
- **Better source:** ReAct OpenReview; Reflexion arXiv; AutoGen arXiv.
- **Note:** skip

## 3. SEO Roundups of "Top AI Agent Frameworks 2025/2026"

- **Type:** SEO roundup
- **Evidence tier:** Reject
- **Reliability:** secondary / low
- **Reason rejected:** They optimize for tool discovery rather than technical claims. Lists are often outdated quickly and do not explain architecture, protocols, evaluation, or safety.
- **Better source:** official framework docs and protocol repositories.
- **Note:** skip

## 4. News Summaries of A2A

- **Examples searched:** summaries of Google's A2A launch and Linux Foundation transfer.
- **Type:** tech news / secondary
- **Evidence tier:** Reject
- **Reliability:** secondary
- **Reason rejected:** They mostly restate Google and Linux Foundation announcements. For protocol scope and project status, the specification and official announcements are stronger.
- **Better source:** Google Developers Blog; A2A latest specification; Linux Foundation A2A v1.0 release.
- **Note:** skip

## 5. Unverified GitHub Awesome Lists for Agents

- **Type:** curated list
- **Evidence tier:** Reject
- **Reliability:** secondary / variable
- **Reason rejected:** Helpful for discovery but not evidence. They do not validate adoption, correctness, or architectural fit.
- **Better source:** official repos, specs, and peer-reviewed papers.
- **Note:** skip

## 6. Wikipedia Pages on Software Agents, Multi-Agent Systems, and Prompt Injection

- **Type:** encyclopedia
- **Evidence tier:** Reject for page claims; possible navigation only
- **Reliability:** secondary
- **Reason rejected:** Useful for finding classic references, but not appropriate as a primary citation when original papers and official specs are available.
- **Better source:** Wooldridge & Jennings; Shoham; Smith; OWASP; AgentDojo.
- **Note:** skip

## 7. Product Marketing Claims About "Autonomous Employees"

- **Examples searched:** agent vendor pages claiming autonomous digital workers or employees.
- **Type:** marketing
- **Evidence tier:** Reject
- **Reliability:** weak / self-promotional
- **Reason rejected:** They often lack implementation detail, evaluation evidence, or clear boundaries around human oversight.
- **Better source:** product docs for feature existence; benchmarks and system cards for capability/safety framing.
- **Note:** skip

## 8. Blog Posts Claiming "MCP Won" or "A2A Failed"

- **Type:** opinion / market analysis
- **Evidence tier:** Reject unless backed by data
- **Reliability:** secondary
- **Reason rejected:** Comparative adoption claims were usually based on impressions, social media, or single proxy metrics. This refresh did not find a robust apples-to-apples adoption study.
- **Better source:** MCP spec/repo, A2A spec, Linux Foundation status updates, and future quantitative adoption evidence if found.
- **Note:** skip

## 9. General RAG Surveys Used as Memory Sources

- **Type:** survey
- **Evidence tier:** Tier 3 at best for this task
- **Reliability:** secondary
- **Reason rejected:** RAG is relevant, but the current page needs agent memory distinctions. RAG surveys risk reinforcing the error that memory equals vector retrieval.
- **Better source:** MemGPT; Generative Agents; Reflexion.
- **Note:** skip for memory typology; maybe for a separate RAG bibliography

## 10. AI-Agent Taxonomy Blog Diagrams Without Citations

- **Type:** secondary diagram / explainer
- **Evidence tier:** Reject
- **Reliability:** weak
- **Reason rejected:** Many diagrams introduce levels, pyramids, or maturity models without evidence and would reintroduce the page's original "ladder" problem.
- **Better source:** Anthropic Building Effective Agents; OpenAI Practical Guide; classic MAS literature.
- **Note:** skip

## 11. Leaderboard Snapshots Without Benchmark Methodology

- **Type:** leaderboard / ranking
- **Evidence tier:** Reject for durable claims
- **Reliability:** time-sensitive
- **Reason rejected:** Scores change quickly and often depend on harness details. The page should cite benchmark design, task families, and evaluation dimensions, not transient rankings.
- **Better source:** AgentBench, WebArena, SWE-bench, GAIA, tau-bench, OSWorld, PaperBench, BrowseComp.
- **Note:** skip for page claims; maybe for local evaluation planning

## 12. Protocol Vendor Comparison Tables

- **Type:** secondary comparison
- **Evidence tier:** Reject
- **Reliability:** secondary / often promotional
- **Reason rejected:** Tables often compare MCP, A2A, ACP, Agent Protocol, and AGNTCY without respecting different scopes. They are useful for discovery but not for taxonomy claims.
- **Better source:** each official spec and repo.
- **Note:** skip
